## 3. Discovered Patterns

- Stale closure anti-pattern in `chat.message` handler — `streamingThoughts` read from render closure instead of ref
- WebSocket lifecycle inconsistency — main page has no reconnect, health/marketplace pages have reconnect but different message formats
- Missing memoization — `renderFormattedContent`, `cleanMessage`, `getAgentMeta`, `handleLaneSwitch` all recreated every render
- Component defined inside component — `CollapsibleThoughts` creates new component every render, losing state
- Inconsistent WS message format across pages — `{session_id}` vs `{request_id, session_id: {id}}`
- No error boundary — single unhandled error crashes entire dashboard
- Hardcoded URLs on main page while settings page uses dynamic hostname
