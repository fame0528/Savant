Statistical Circuit Breaker (savant_echo::circuit_breaker)  - Monitoring: Tracks success/failure metrics of every tool execution. - Rollback: Automatically triggers a registry rollback to the previous stable epoch if the error rate exceeds a heuristic threshold (default 5%).  ## Autonomous Workflow  1. Refactor: An agent identifies a performance bottleneck or missing feature. 2. Compile: The EchoCompiler generates a specialized WASM plugin. 3. Verify: Automated test suites validate the new plugin in an isolated environment. 4. Swap: The HotSwappableRegistry promotes the plugin to the active epoch.  ## Safety Guards  - Memory/Fuel Limits: All WASM plugins run with strictly enforced memory (max 64MB) and fuel (max 10M instructions) limits. - Stateless Verification: Tool signatures are verified using Ed25519 SecurityEnclave before execution.

use arc_swap::ArcSwap;
use std::sync::Arc;
use std::collections::HashMap;

# [derive(Clone, Debug)]

pub struct Tool {
    pub name: String,
    pub version: u64,
    pub wasm_module: Vec<u8>,
    pub signature: ed25519::Signature, // Over `wasm_module` hash
}

# [derive(Clone, Debug)]

pub struct Epoch {
    pub tools: HashMap<String, Arc<Tool>>,
    pub epoch_id: u64, // Monotonic, WAL-logged
}

pub struct EchoRegistry {
    current_epoch: ArcSwap<Epoch>,
    previous_epoch: Option<Arc<Epoch>>, // For rollback
    circuit_breaker: Arc<CircuitBreaker>,
    security_enclave: Arc<SecurityEnclave>,
    wal: Arc<dyn WalSink>, // Abstract persistence layer
}

impl EchoRegistry {
    /// Creates a new empty registry. WAL entry: `REGISTRY_INIT`.
    pub fn new(
        security_enclave: Arc<SecurityEnclave>,
        circuit_breaker: Arc<CircuitBreaker>,
        wal: Arc<dyn WalSink>,
    ) -> Result<Arc<Self>, Error> {
        let epoch = Epoch {
            tools: HashMap::new(),
            epoch_id: 0,
        };
        let reg = Arc::new(Self {
            current_epoch: ArcSwap::from_pointee(epoch),
            previous_epoch: None,
            circuit_breaker,
            security_enclave,
            wal,
        });
        wal.log(&WalEntry::RegistryInit { epoch_id: 0 })?;
        Ok(reg)
    }

    /// Atomic epoch swap with pre-validation and WAL commitment.
    pub fn try_swap_epoch(self: &Arc<Self>, new_epoch: Epoch) -> Result<(), Error> {
        // 1. Verify all tool signatures in the new epoch.
        for tool in new_epoch.tools.values() {
            self.security_enclave.verify(&tool.wasm_module, &tool.signature)?;
        }

        // 2. Log intent to WAL ( LAW: WAL is Law ).
        let old_epoch = self.current_epoch.load();
        self.wal.log(&WalEntry::EpochSwapIntent {
            old_epoch_id: old_epoch.epoch_id,
            new_epoch_id: new_epoch.epoch_id,
            tool_names: new_epoch.tools.keys().cloned().collect(),
        })?;

        // 3. Atomically swap the pointer.
        let old_epoch = self.current_epoch.swap(Arc::new(new_epoch.clone()));
        self.previous_epoch = Some(old_epoch);

        // 4. Log committed swap.
        self.wal.log(&WalEntry::EpochSwapped {
            epoch_id: new_epoch.epoch_id,
        })?;

        // 5. Reset circuit breaker for fresh metrics.
        self.circuit_breaker.reset();
        Ok(())
    }

    /// Returns a `ToolHandle` that auto-reports to circuit breaker.
    /// `self: &Arc<Self>` allows `ToolHandle` to hold `Weak<Self>` for rollback.
    pub fn get_tool(self: &Arc<Self>, name: &str) -> Option<ToolHandle> {
        let epoch = self.current_epoch.load();
        let tool = epoch.tools.get(name)?.clone();
        Some(ToolHandle {
            tool,
            circuit_breaker: self.circuit_breaker.clone(),
            registry: Arc::downgrade(self),
            tool_name: name.to_string(),
        })
    }

    /// Manual rollback (also called by circuit breaker).
    pub fn rollback(&self) {
        if let Some(prev) = self.previous_epoch.take() {
            let _ = self.current_epoch.swap(prev);
            self.circuit_breaker.reset();
            let _ = self.wal.log(&WalEntry::EpochRollback {
                epoch_id: self.current_epoch.load().epoch_id,
            });
        }
    }
}

use wasmtime::{Engine, Store, Module, Instance, Memory, Func, ResourceLimiter};
use std::sync::Arc;

pub struct ToolHandle {
    tool: Arc<Tool>,
    circuit_breaker: Arc<CircuitBreaker>,
    registry: std::sync::Weak<EchoRegistry>,
    tool_name: String,
}

impl ToolHandle {
    /// Executes the WASM tool with strict resource limits.
    /// Protocol: WASM module exports `(memory "memory")`, `(func $run (param i32 i32) (result i32 i32))`.
    /// Input: pointer+len into WASM memory. Output: pointer+len from WASM memory (allocated by module's malloc).
    /// Memory management: The module must export `(func $free (param i32 i32))` to avoid leaks.
    pub fn run(&self, input: &str) -> Result<String, Error> {
        // 1. Configure engine with hard limits.
        let mut config = Engine::default().config();
        config
            .with_limits(Some(64 *1024* 1024), Some(10_000_000))?; // 64MB memory, 10M fuel
        let engine = Engine::new(&config)?;

        // 2. Instantiate module.
        let mut store = Store::new(&engine, ());
        store.set_fuel(10_000_000)?;
        let module = Module::new(&engine, &self.tool.wasm_module)?;
        let instance = Instance::new(&mut store, &module, &[])?;

        // 3. Get memory and allocation functions.
        let memory = instance
            .get_memory(&mut store, "memory")
            .ok_or(Error::MissingExport("memory"))?;
        let alloc = instance
            .get_typed_func::<i32, i32>(&mut store, "alloc")
            .ok_or(Error::MissingExport("alloc"))?;
        let free = instance
            .get_typed_func::<(i32, i32), ()>(&mut store, "free")
            .ok_or(Error::MissingExport("free"))?;

        // 4. Copy input into WASM memory.
        let input_bytes = input.as_bytes();
        let input_len = input_bytes.len() as i32;
        let input_ptr = alloc.call(&mut store, input_len as i32)?;
        memory.write(&mut store, input_ptr as usize, input_bytes)?;

        // 5. Call the tool's `run` function.
        let run = instance
            .get_typed_func::<(i32, i32), (i32, i32)>(&mut store, "run")
            .map_err(|_| Error::MissingExport("run"))?;
        let (output_ptr, output_len) = run.call(&mut store, (input_ptr, input_len))?;

        // 6. Read output.
        let output_bytes = memory.read(&mut store, output_ptr as usize, output_len as usize)?;
        let output = String::from_utf8(output_bytes.to_vec()).map_err(|_| Error::Utf8)?;

        // 7. Free the output buffer (avoid memory leak).
        free.call(&mut store, (output_ptr, output_len))?;

        // 8. Record metric & check circuit breaker.
        self.circuit_breaker.record_success(&self.tool_name);
        if self.circuit_breaker.should_rollback() {
            if let Some(reg) = self.registry.upgrade() {
                reg.rollback();
            }
        }

        Ok(output)
    }
}

2.3 Circuit Breaker (circuit_breaker.rs)

use std::sync::{Arc, Mutex};
use std::collections::HashMap;

# [derive(Debug, Default)]

struct Metrics {
    total: u64,
    failures: u64,
}

pub struct CircuitBreaker {
    metrics: Mutex<HashMap<String, Metrics>>,
    threshold: f64,      // e.g., 0.05 for 5%
    min_samples: u64,    // e.g., 100 to avoid flapping
}

impl CircuitBreaker {
    pub fn new(threshold: f64, min_samples: u64) -> Self {
        Self {
            metrics: Mutex::new(HashMap::new()),
            threshold,
            min_samples,
        }
    }

    pub fn record_success(&self, tool: &str) {
        let mut metrics = self.metrics.lock().unwrap();
        let m = metrics.entry(tool.to_string()).or_default();
        m.total += 1;
    }

    pub fn record_failure(&self, tool: &str) {
        let mut metrics = self.metrics.lock().unwrap();
        let m = metrics.entry(tool.to_string()).or_default();
        m.total += 1;
        m.failures += 1;
    }

    pub fn should_rollback(&self) -> bool {
        let metrics = self.metrics.lock().unwrap();
        for m in metrics.values() {
            if m.total >= self.min_samples {
                let rate = m.failures as f64 / m.total as f64;
                if rate > self.threshold {
                    return true;
                }
            }
        }
        false
    }

    pub fn reset(&self) {
        let mut metrics = self.metrics.lock().unwrap();
        metrics.clear();
    }
}

    // 2. Landlock sandbox: allow only project dir + /tmp.

        landlock::Rule::new()
            .allow(cwd_fd.clone())
            .allow(Access::Read | Access::Write | Access::Execute),

    let tmp_fd = PathFd::create(tmp, Access::Execute)?;

        landlock::Rule::new()
            .allow(tmp_fd)
            .allow(Access::Read | Access::Write | Access::Execute),

1. Pre‑Warmed Agent Provisioning: The Solution

3.1 Design Principles

Allocate Once, Activate Many – All memory, tokens, and personality data are prepared while the system is idle (during hot‑reload).

Cache‑Line Alignment – Stack and heap arenas placed on separate 64‑KiB pages to avoid false sharing and ensure L1/L2 hit rates.

Shared Read‑Only Blobs – Personality data lives in a single, mapped Nexus segment; all agents reference the same physical pages.

Synchronized Flip – Activation is a single store‑release to a global activation_flag; all agents spin on a load‑acquire and start simultaneously.

Atomic Intent Logging – The entire provisioning plan is logged to WAL as one transaction before any memory is touched.

3.2 Memory Layout

Nexus Shared Memory (4 GB reserved, currently 1 GB mapped)
├─ Global Control Block (1 page)
│   └─ activation_flag: AtomicUsize
├─ Agent Arena Table (101 entries × 64 bytes)
│   └─ per‑agent: stack_base, stack_size, heap_base, heap_size, personality_offset, cct_offset
├─ Pre‑Warmed Stacks (fixed 2 MiB each, 64 KiB aligned, touched)
├─ Pre‑Warmed Heaps (bump‑allocator arenas, sizes per agent type)
├─ Personality Blob Region (read‑only, shared)
│   ├─ Common personality templates
│   └─ Agent‑specific overrides
└─ CCT Cache Region (bloom filters, one per agent)

3.3 Activation Flow

Phase 0 – Planning (single‑threaded, before hot‑reload)

Determine which 10 agents to activate (based on role, priority).

For each agent, compute its arena requirements.

Reserve slots in the Agent Arena Table.

Write an Intent::Provision(agents: Vec<AgentSpec>) record to WAL and fsync.

Phase 1 – Pre‑Warm (parallel, during hot‑reload)

For each agent in the plan:

Allocate stack pages, write to each page (guarantee residency).

Initialize heap arena bump pointer to zero, optionally pre‑fill with common objects.

Copy personality data from read‑only blob into agent‑local view (still same physical pages).

Compute CCT bloom filter; store in designated region.

Fill its Arena Table entry with pointers and offsets.

Ensure all memory writes are visible (full memory barrier).

Phase 2 – Synchronized Activation

Store the number of agents to activate into activation_flag using Ordering::Release.

All pre‑warmed agents, already in their “spinning” state (running a tiny loop waiting for the flag), observe the change with Ordering::Acquire and jump to their main entry point.

Activation latency ≈ one cache‑coherent write + one read, i.e., < 100 ns on modern NUMA‑aware hardware.

Current Limits:

Single Point of Failure: One WAL file per agent. If that file corrupts (bitrot, filesystem bug), we lose that agent's memory chain. No replication.

No Integrity Verification: We rely on the filesystem's checksums. No internal Merkle proof or per-entry checksum beyond the operation_hash itself (which is an output, not a guard).

Recovery Time: To replay 1M entries takes ~2 seconds on my current hardware. That's downtime we can't afford during a hot restart.

Size Limits: No hard cap. The file grows indefinitely. Eventually we'll hit filesystem limits (16TB on ext4) or performance degradation due to fragmentation.


Why This Design?


Simplicity. TheZeroClaw synthesis prioritized minimalism.

O_DIRECT + fsync gives us durability with minimal kernel involvement.

One file = no directory lookup overhead.

Weak Points (The Cracks)



Fsync Bottleneck: Every intent hits the storage device's sync queue. This is the single biggest latency contributor for state changes. At our target 100µs end-to-end, fsync alone costs ~50-200µs on consumer NVMe. That's unacceptable for a sub-millisecond system.



No Redundancy: The WAL is a single point of truth. If the file is lost (accidental delete, filesystem corruption, malicious actor with write access), that agent's memory is gone forever. The Nexus Bridge can't reconstruct it.



No Proactive Health Monitoring: We don't continuously verify the WAL's integrity. We only notice corruption on replay failure, which is too late.



Unbounded Growth: No archival or compaction strategy. Old intents remain. We eventually need to prune or snapshot and truncate, but doing so risks breaking the "WAL is Law" principle if not done perfectly.



No Backpressure: If the storage device can't keep up, the agent blocks on fsync. There's no queue or adaptive throttle. This could cascade into swarm-wide heartbeat failures.



Checkpoint Overhead: Snapshotting every 10k entries requires a global state freeze for ~50ms. That's a noticeable pause in a 24/7 system.
